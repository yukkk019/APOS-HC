# newneed（ニーズ領域抽出・看護計画）操作手順書

## 1. 概要

**newneed** は、アセスメント票のデータ（CSV／DB）に基づき、**23項目（N01～N23）のニーズ領域**のうち「値が立った」項目だけを抽出し、一覧表示するモジュールです。あわせて**看護計画書**の入力・保存・PDF出力ができます。

- **主な機能**
  - ニーズ領域の抽出表示（事業所番号・個人番号・回次で指定）
  - 看護計画の記入・保存（ニーズ領域ごと）
  - 看護計画のPDF出力

---

## 2. 画面へのアクセス方法

### 2.1 前提条件

- バックエンド（FastAPI）が起動していること
- ブラウザで次のいずれかのURLにアクセスできること

### 2.2 表示画面のURL

| 用途 | URL |
|------|-----|
| ニーズ領域表示画面（メイン） | `http://<サーバー>/api/newneed/page` または `http://<サーバー>/api/newneed/static/display.html` |
| 看護計画画面 | ニーズ領域表示画面から「看護計画を書く」をクリックすると遷移（URLに `care-plan.html` が含まれる） |

**例（ローカル）:**  
`http://localhost:8000/api/newneed/page`

## 3. ニーズ領域の表示（手順）

### 3.1 手順

1. ブラウザで **ニーズ領域抽出システム** の画面を開く（上記URL）。
2. 次の項目を入力する。
   - **事業所番号** … 例: `A001`
   - **個人番号** … 例: `0001`
   - **回次** … 「1回目」「2回目」「3回目」のいずれか
3. **「表示」** ボタンをクリックする。
4. 該当する利用者・回次のデータが存在すれば、**値が立ったニーズ領域（N01～N23）のみ**が一覧表示される。
5. 画面上部の **目次（抽出されたニーズ領域）** のボタンをクリックすると、該当するニーズ領域のブロックへスクロールする。

### 3.2 表示結果の見方

- **該当した項目のみ**が表示される（該当なしのニーズ領域は表示されない）。
- 各ニーズ領域ブロックには次の情報が出る。
  - **ニーズID**（例: N01, N02）
  - **ニーズ名称**（日本語）
  - **該当内容**（アセスメントの質問・回答に相当する表示文言の一覧）
- 各ブロック右側の **「看護計画を書く」** をクリックすると、そのニーズ領域用の看護計画画面に遷移する。

### 3.3 エラー・注意

- **事業所番号・個人番号が未入力** … 「事業所番号と個人番号を入力してください。」と表示される。
- **レコードが見つからない** … 指定した「事業所番号_個人番号」と回次に対応するデータがDBまたはCSVにない場合、「レコードが見つかりません」と表示される。
- **該当項目が一つもない** … 「該当した項目はありません。」と表示される。

## 4. 看護計画の記入・保存（手順）

### 4.1 看護計画画面の開き方

1. **ニーズ領域表示画面**で、対象のニーズ領域ブロックの **「看護計画を書く」** をクリックする。
2. 遷移先の **看護計画書** 画面で、左側に「該当内容（参照）」、右側に看護計画の入力フォームが表示される。

### 4.2 看護計画の記入手順

1. 左側の **該当内容** を参照しながら、右側のフォームに入力する。
2. 入力項目（概要）:
   - **看護目標（計画1～。複数可）**
   - **具体的なケア内容**
   - **支援方法**（チェック等）
   - **前回からの変化**（レベル・内容・自由記述）
   - 必要に応じて **「＋ 計画を追加」** で計画を複数追加
3. **「保存」** ボタンをクリックして保存する。
4. 保存成功時は「看護計画を保存しました」などのメッセージが表示される。

### 4.3 PDF出力

- **「PDF」** ボタンをクリックすると、現在表示している看護計画の内容を元にPDF用の印刷ダイアログが開く。
- ブラウザの印刷機能で「PDFに保存」を選べば、PDFファイルとして保存できる。

### 4.4 ニーズ領域一覧へ戻る

- **「← ニーズ領域一覧に戻る」** のリンクをクリックすると、ニーズ領域表示画面（display.html）に戻る。

## 5. API（開発者・連携用）

システム内や他ツールから利用する場合の代表的なAPIは以下のとおりです。

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/newneed/` | モジュール状態確認（`{"module":"newneed","status":"ok"}`） |
| GET | `/api/newneed/display` | 23項目表示用JSON取得 |
| GET | `/api/newneed/page` | 表示画面（display.html）へリダイレクト |
| GET | `/api/newneed/care-plan/form` | 看護計画フォーム用データ取得 |
| POST | `/api/newneed/care-plan` | 看護計画の保存 |

### 5.1 表示データ取得: GET `/api/newneed/display`

**クエリパラメータ**

| パラメータ | 必須 | 説明 |
|------------|------|------|
| `user_id` | ※1 | ユーザーID（`事業所番号_個人番号` の形式） |
| `office_id` | ※1 | 事業所ID（`personal_id` とセットで指定時） |
| `personal_id` | ※1 | 個人ID（`office_id` とセットで指定時） |
| `round` または `round_num` | 任意 | 評価回次（1/2/3）。省略時は 1 |

※1 `user_id` を指定するか、`office_id` と `personal_id` の両方を指定してください。

**レスポンス例**

```json
{
  "user_id": "A001_0001",
  "assessment_round": "1",
  "items": [
    {
      "need_id": "N01",
      "need_name_jp": "収入／家計／生活苦",
      "contents": [
        { "column": "economic_status_1_2", "display_text": " 1. この1年で家計に困ったことは？ 年２~３回" }
      ]
    }
  ]
}
```

## 6. データ・設定ファイル（管理者向け）

newneed の表示内容は、次のCSV・スキーマで制御されています。

### 6.1 フォルダ構成（抜粋）

```
backend/app/newneed/
├── api_router.py           # API・表示ロジック
├── column_display_text.csv # カラムごとの表示文言
├── sync_display_text_to_schema.py  # 表示文言をスキーマに同期するスクリプト
├── spec/
│   ├── needs_master_schema.csv   # ニーズマスター（N01～N23の名称など）
│   └── need_rules_schema.csv     # ニーズ判定ルール（どのカラムがどのニーズに紐づくか）
└── static/
    ├── display.html        # ニーズ領域表示画面
    ├── care-plan.html      # 看護計画画面
    ├── display.css         # 共通スタイル
    └── html2pdf.bundle.min.js  # PDF用
```

### 6.2 表示文言の変更（column_display_text.csv）

- **column_display_text.csv** で、`need_id`・`column_name` に対応する **display_text** を編集すると、画面上の「該当内容」の文言が変わります。
- 1行の形式: `need_id,column_name,display_text`
- 文言や半角スペースはCSVの記載がそのまま使われます。

### 6.3 表示文言をスキーマに反映する（sync_display_text_to_schema.py）

- **column_display_text.csv** の内容を **spec/need_rules_schema.csv** の `confidence_note` に反映したい場合に使用します。
- 実行方法（プロジェクトルートで）:
  ```bash
  python -m app.newneed.sync_display_text_to_schema
  ```
- 同一の `(need_id, column_name)` が複数行ある場合は、**最後の行の display_text** が使われます。

### 6.4 ニーズマスターの変更（needs_master_schema.csv）

- **spec/needs_master_schema.csv** で、N01～N23 の **need_name_jp**（日本語名称）などを変更できます。
- カラム例: `need_id`, `need_category`, `need_name_jp`, `need_name_en`, `description`, `source_section`, `notes`

### 6.5 ニーズ判定ルールの変更（need_rules_schema.csv）

- **spec/need_rules_schema.csv** で、どのアセスメント項目（カラム）がどのニーズに紐づくかを定義しています。
- 主なカラム: `need_id`, `if_column`, `operator`, `target_value`, `confidence_note`
- **operator** の例: `==`（一致）, `>=`, `<=`, `between`, `contains`, `!=`（値があるか）など
- ルールを変更した場合は、**APIを提供しているサーバーを再起動**すると反映されます（起動時にCSVを読み込みます）。

## 7. トラブルシューティング

| 現象 | 確認すること |
|------|----------------|
| 画面が表示されない | バックエンドが起動しているか、URLが `/api/newneed/page` または `/api/newneed/static/display.html` になっているか確認 |
| 「レコードが見つかりません」 | 指定した事業所番号・個人番号・回次のデータがDBまたはCSVに存在するか確認 |
| 該当項目が表示されない | アセスメントデータの該当カラムに値が入っているか、`need_rules_schema.csv` のルール（if_column, operator, target_value）が正しいか確認 |
| 表示文言が想定と違う | `column_display_text.csv` の `display_text` を確認・修正。必要なら `sync_display_text_to_schema.py` を実行 |
| 看護計画が保存されない | ネットワークエラーやサーバーエラーがないか確認。POST先が `/api/newneed/care-plan` であること、必須項目（user_id, assessment_round, need_id）が送られているかを確認 |
